§LLM Provider Configuration (Core)

This module contains the core LLM provider functionality without any CLI dependencies. It provides the minimal interface needed by the core LLM functionality using pure configuration types.

§Architecture

This module is part of the core layer and contains only business logic. All configuration, validation, and external dependencies are handled by the higher-level configuration layer.

§Examples

use rustree::core::llm::{CoreLlmConfig, CoreLlmProvider, LlmConfig};
use std::time::Duration;

// Create core configuration (normally done by config layer)
let core_config = CoreLlmConfig {
    provider: CoreLlmProvider::OpenAi,
    model: "gpt-4".to_string(),
    api_key: "sk-...".to_string(),
    endpoint: None,
    temperature: 0.7,
    max_tokens: 1000,
    timeout: Duration::from_secs(60),
};

// Create LLM config for core operations
let config = LlmConfig::new(core_config);
